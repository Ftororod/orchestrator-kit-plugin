# Pain points — <PROYECTO>

Fricciones, quejas y cosas que duelen del proceso o del stack. Lo que se repite
aqui es candidato a leccion de napkin o a decision del PO.

Formato:

## <area / tema>

- **Friccion:** <que duele>
- **Frecuencia:** <una vez / recurrente>
- **Estado:** abierta / mitigada / resuelta (<fecha>)
- **Nota:** <workaround actual o por que sigue abierta>
